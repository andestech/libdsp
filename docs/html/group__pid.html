Detailed Description

PID (Proportional-Integral-Derivative) controller functions provide a continuous-loop feedback mechanism to control output errors according to the inputs periodically. The state of PID controller is shown in the following diagram:

Figure 3. State of PID Controller

At the controller process when current data is input each time, the behavior of PID controller is as follows:

Andes DSP library supports distinct PID initialization and PID functions for the following data types: floating-point, Q31 and Q15. These functions are introduced in the subsections below.

)

PID control of floating-point input.

Parameters
[in]*instancepointer of the instance structure
[in]srccurrent data
Returns
Output of controller

Note:

The riscv_dsp_init_pid_f32 function must be called to initialize the controller first, then the riscv_dsp_pid_f32 function can be executed to do loop process of adjusting the controller’s outputs with the current data. Please see in example.

Example 

     riscv_dsp_pid_f32_t instance;
     instance.Kp = 0.02F;
     instance.Ki = 0.03F;
     instance.Kd = 0.014F;
     //initialize controller
     riscv_dsp_init_pid_f32(&instance, 1);
     //Begin controller
     while (1)
     {
         f32_t output;
         f32_t input = … read data from input port …
         output = riscv_dsp_pid_f32(&instance, input);
         …
     }
This example also serves as a reference for examples of Q31 or Q15 PID functions.

)

PID initializatopn control function of floating-point formats.

Parameters
[in]*instancepointer of the instance structure
[in]setdo nothing if its value is zero; otherwise, clear the state buffer.
Returns
none.

Note:

  1. This function should be called to initialize the controller before the function riscv_dsp_pid_f32 is called.
  2. While parameters gain1, gain2, gain3 and state[3] are managed by the controller, you have to input Kp, Ki and Kd on your own.
